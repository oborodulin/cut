TestsInfo			030	1CON	Процеcc тестирования в основном состоит из написания тестовых сценариев. TestsResultsInfo	031	1CON	При этом, тест выполнен успешно ^({V1}^), если пройдены проверки всех тестовых случаев тестового сценария. Тест {V3} - если тест вернул соответствующий код возврата, например, в случае, когда какой-либо тестовый сценарий пока не реализован. Тест с ошибкой ^({V4}^) - если тест обнаружил неверный параметр, проблемы с сетью, отсутствующий объект файловой системы ^(файл, каталог, ссылку, ...^), отсутствующую утилиту, необходимую для работы теста и т.д.
TestFunc2			032	1CON	2. Тестирование 'чёрного' и 'белого ящика'
TestsPrefixInfo		033	1CON	Для любых исполняемых модулей поддерживается тестирование 'чёрного ящика', а для командных сценариев ещё и 'белого ящика', в зависимости от префикса имени тестового сценария. Префикс {V1} - для тестирования 'чёрного ящика' ^(например, {V1}script.cmd^), а {V2} - для 'белого ящика' ^(например, {V2}script.cmd^). Метод 'чёрного ящика' можно применять для тестирования интерфейса командной строки исполняемых модулей, а 'белого ящика' - только для непосредственного тестирования процедур командных сценариев, так как перед выполнением теста исходный сценарий автоматически присоединяется к тестовому.

Определяется время выполнения тестирования, в том числе по каждому тестовому набору и каждому успешно выполненному тесту в наборе.

Например, сценарий {V1}src_script\{V2} выполнится перед выполнением тестового набора для исходного сценария src_script.cmd, а сценарий {V1}src_script\{V3} - после выполнения тестового набора.
